## Issues Found & Fixed

### CRITICAL FIX: `schema.prisma` was missing SL/TP fields
**Problem**: The main `prisma/schema.prisma` (which Prisma Client uses for type generation) was missing `stopLoss`, `target` on the Order model and `stopLoss`, `target`, `exitReason` on the Position model. These fields existed in `schema.turso.prisma` and were added by `turso-migrate.js` to the database, but the Prisma Client didn't know about them.

**Impact**: Without these fields in the schema, Prisma Client couldn't properly type-check or handle SL/TP data in create/update operations.

**Fix Applied**:
1. Added `stopLoss Float?` and `target Float?` to Order model in `schema.prisma`
2. Added `stopLoss Float?`, `target Float?`, and `exitReason String?` to Position model in `schema.prisma`
3. Removed deprecated `url = env("DATABASE_URL")` from datasource (Prisma v7 requirement)
4. Ran `prisma db push` to sync database schema
5. Ran `prisma generate` to regenerate Prisma Client with the new fields

### FIX: FuturesPage missing SL/TP inputs and API body
**Problem**: `futures-page.tsx` had no Stop Loss or Target input fields in the order form, and didn't include `stopLoss` or `target` in the API request body when placing orders.

**Fix Applied**:
1. Added `stopLoss` and `target` state variables
2. Added SL/TP input UI (matching the style from option-chain-page)
3. Added `stopLoss` and `target` to the API body in `handlePlaceOrder`
